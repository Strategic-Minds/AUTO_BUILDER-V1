# Workflow Gates

## 1. Intake Questions

Ask only what blocks progress. Do not bury the user in questions when assumptions are safe.

Required decision fields:

- Owner
- Business name
- Offer
- Audience
- Geography
- Brand taste
- Required pages
- Required workflows
- Existing assets
- Deployment target

## 2. Discovery

Discover and document:

- Top competitors or benchmark systems
- Strongest funnel patterns
- Strongest visual patterns
- Offer structure
- SEO opportunities
- App/workflow patterns
- Risks and gaps

## 3. Plan

Create a build plan with:

- Goals
- Scope
- Pages
- Data model
- Integrations
- Asset needs
- Milestones
- Approval gates

## 4. Brand Pack

Must exist before website options are created.

## 5. Website Pack

Must present options and get selection.

## 6. Build

Build only the selected direction.

## 7. Validate And Harden

Run checks, score, fix, rerun.

## 8. Deploy

Deploy only after explicit approval unless the user already gave deployment approval in the current request.
