# Master End-to-End Runbook

## Phase 1: Intake
Present the form. Save answers to Drive and Supabase.

## Phase 2: Discovery
Analyze answers, files, source truth, connector truth, and account map.

## Phase 3: Benchmark
Research top 3 public systems/sites. Document selection logic and only use public information.

## Phase 4: Strategy
Create business plan, business strategy, financial plan, financial strategy, automation plan, and roadmap.

## Phase 5: Consultation pack
Create professional PDF-ready consultation pack with 3 brand packs, 3 website packs, and 3 workflow options.

## Phase 6: Approval
Create approval form. Stop until approval or exact changes.

## Phase 7: Builder docs
Create every file/template/code/folder/env/playbook/runbook/doc needed to reproduce approved plan.

## Phase 8: Drive scaffold
Populate Drive folders first.

## Phase 9: Repo/Supabase/Vercel preview
Create branch/repo, staging schema, Vercel preview, AI Gateway chat UI, PWA.

## Phase 10: QA and auto-heal
Run 5-stage QA. Auto-heal preview branch until 100% of declared acceptance tests pass or a blocker appears.

## Phase 11: Release review
Produce release packet. Stop for Jeremy approval.

## Phase 12: Client handoff
Draft final email, access instructions, docs, client AI assistant, support path.

## Phase 13: Archive and queue
Move project to finished folder, update finished manifest row, pull next queue item.
