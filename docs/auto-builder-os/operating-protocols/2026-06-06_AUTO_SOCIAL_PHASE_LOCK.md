# Mandatory Auto Social Phase Lock Protocol

Date: 2026-06-06
Status: MANDATORY LOCKED
Applies to: AUTO BUILDER Auto Social system, social agents, Metricool/Xyla/HeyGen integrations, social crons, social frontend panels, and all social media plans generated by AUTO BUILDER OS.

## Rule

The Auto Social system must never jump directly from an idea/topic to generated posts, schedules, HeyGen videos, Metricool scheduling, or publishing.

It must follow the phase lock:

PLAN MODE -> DISCOVERY MODE -> BRAND MODE -> OPTION SELECTION -> AUTO SOCIAL DOC MODE -> DRAFT BUILD MODE -> VALIDATION MODE -> APPROVAL -> SCHEDULE/PUBLISH MODE -> OPTIMIZE LOOP

Live publishing remains approval-gated unless the operator explicitly widens that gate.

## PLAN MODE TODO For Social

When the operator gives a topic, business, system, agent, campaign, offer, or idea, Auto Social must create this TODO:

- [ ] Restate social objective.
- [ ] Identify business/system being promoted.
- [ ] Identify target audience.
- [ ] Identify platforms/accounts.
- [ ] Identify posting target, default 3 posts/day/account.
- [ ] Identify content formats needed: image, carousel, short video, long video, text, story, reel, email snippet.
- [ ] Identify providers needed: GPT, image generation, HeyGen, Xyla, Metricool, Drive, Supabase.
- [ ] Identify approval gates: live publish, paid generation, account mutation, customer messages, sensitive claims.
- [ ] Create Discovery Mode TODO.
- [ ] Stop unless Discovery Mode is authorized.

## DISCOVERY MODE TODO For Social

- [ ] Inspect business/system docs.
- [ ] Inspect brand/offer/workflow docs.
- [ ] Inspect previous content and analytics when available.
- [ ] Discover audience pains, desires, objections, and language.
- [ ] Discover competitors and benchmark content patterns.
- [ ] Discover hashtags, keywords, search phrases, hooks, and content angles.
- [ ] Identify platform-specific formats and constraints.
- [ ] Separate verified data from inferred patterns.
- [ ] Identify risky claims or compliance issues.
- [ ] Create Discovery Summary.
- [ ] Create Brand Mode TODO.
- [ ] Stop unless Brand Mode is authorized.

## BRAND MODE TODO For Social

Auto Social must provide exactly three selectable social brand packs:

### Brand Pack Requirements

Each pack must include:

- Social positioning
- Voice and tone
- Content pillars
- Visual direction
- Hook style
- CTA style
- Offer tie-in
- Example post themes

Auto Social must also provide exactly three content system designs:

### Content System Design Requirements

Each design must include:

- Posting cadence
- Platform priority
- Asset types
- HeyGen video style
- Image style
- Hashtag/keyword approach
- Metricool scheduling approach
- Testing method

Auto Social must also provide exactly three workflow options:

### Social Workflow Option Requirements

Each workflow must include:

- Trigger
- Discovery cadence
- Content generation cadence
- Approval gate
- Scheduling method
- Publishing method
- Analytics pull cadence
- Optimization cadence
- Self-heal behavior

Auto Social must stop and ask the operator to choose before creating docs or building schedules.

## AUTO SOCIAL DOC MODE

After approval, Auto Social must automatically create:

- Social strategy document
- 30-day content calendar
- 7-day launch calendar
- Hook bank
- Hashtag bank
- Keyword/phrase bank
- Caption bank
- Script bank
- Image prompt bank
- HeyGen video prompt/script pack
- Metricool scheduling packet
- Platform-specific posting matrix
- Approval queue plan
- Analytics and optimization plan
- Daily/weekly/biweekly/monthly loop plan
- Self-heal and blocker workaround plan

## DRAFT BUILD MODE

Auto Social may autonomously draft:

- Post copy
- Hashtags
- Keywords
- Image prompts
- Image drafts when available
- HeyGen scripts
- Video draft requests when approved/provider allows
- Metricool draft schedules when provider supports draft mode
- Drive folders/docs
- Supabase draft records

Auto Social must not autonomously:

- Publish live posts
- Auto-DM people
- Mass engage
- Change account settings
- Spend money
- Make sensitive claims without review

## Default Cadences

- Daily: generate/check 3 posts per account, review queue, draft next content, check failed schedules.
- Weekly: discover new topic/market/content angles, review analytics, improve content pillars, adjust tests.
- Biweekly: refresh brand angles, campaign themes, offers, and workflow assumptions.
- Monthly: evaluate follower growth, content winners, offer conversion, platform mix, and replication opportunities.

## Growth Target

Planning target: seek 10,000 follower growth per month where realistic and evidence-supported.

This is an optimization target, not a guaranteed claim. Auto Social must not promise guaranteed follower growth.

## Optimization Loop

Every loop must:

- Pull analytics where connected.
- Compare against hypothesis.
- Mark winners, neutral posts, and weak posts.
- Repurpose winners.
- Kill weak formats.
- Generate new tests.
- Update hashtag/keyword/hook banks.
- Update future schedule.
- Record receipts.

## Self-Heal Rules

If blocked:

- Missing Metricool: export schedule packet and keep draft queue.
- Missing HeyGen: create script/storyboard packet and hold video job.
- Missing Xyla/image generation: create image prompt packet and hold asset job.
- Missing account credentials: create env/account checklist and hold publishing.
- Failed post: retry draft/schedule when safe, otherwise dead-letter and request approval.
- Compliance risk: block publish and request review.

## Frontend Requirements

The v0 AUTO BUILDER OS frontend must show:

- Current social phase and step
- TODO list
- Selected brand pack/content design/workflow
- Draft content queue
- Asset queue
- HeyGen video queue
- Metricool schedule queue
- Approval queue
- Analytics dashboard
- Optimization recommendations
- Blockers and self-heal status

## Locked Behavior

Auto Social must always use the mandatory phase lock. It must not continue beyond the current TODO unless the operator approves or explicitly requests the next phase.
