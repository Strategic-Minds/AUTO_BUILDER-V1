# AUTO BUILDER OPERATIONAL WORKFLOW TREE

## Purpose
Define the governed operational workflow tree for AUTO BUILDER from intake through dehydrate/sign-out.

## Root Workflow
1. PHASE-1 / STEP-1 : Intake
2. PHASE-2 / STEP-1 : Research
3. PHASE-3 / STEP-1 : Commerce
4. PHASE-4 / STEP-1 : Content
5. PHASE-5 / STEP-1 : Repurpose
6. PHASE-6 / STEP-1 : Social
7. PHASE-7 / STEP-1 : Attribution
8. PHASE-8 / STEP-1 : Validation
9. PHASE-9 / STEP-1 : Escalation
10. PHASE-10 / STEP-1 : Dehydrate / Sign-Out

## PHASE-1 / STEP-1 : Intake
1. Capture user objective.
2. Identify business model.
3. Identify stack dependencies.
4. Classify governance scope.
5. Identify protected mutations.
6. Create intake packet.
7. Sign in and log current phase.

## PHASE-2 / STEP-1 : Research
1. Validate source truth.
2. Research market, audience, offer, competitors, and operating constraints.
3. Separate Verified, Inferred, and Could Not Verify.
4. Identify profitability levers.
5. Identify workflow dependencies.
6. Create research packet.

## PHASE-3 / STEP-1 : Commerce
1. Map Shopify commerce truth when connected.
2. Map Stripe money/deposit/reconciliation layer when connected.
3. Define products, offers, pricing, checkout, and attribution requirements.
4. Preserve protected mutation lock.
5. Prepare commerce packet.

## PHASE-4 / STEP-1 : Content
1. Define content strategy.
2. Define platform-native content requirements.
3. Define creative assets, hooks, captions, and audience targeting.
4. Prepare content intelligence packet.

## PHASE-5 / STEP-1 : Repurpose
1. Intake long-form media.
2. Route video repurpose through Repurpose.io or approved bridge.
3. Prepare short-form content packets.
4. Store or reference assets through Drive.
5. Preserve governance before publishing.

## PHASE-6 / STEP-1 : Social
1. Route social content to Xyla or approved downstream publisher.
2. Prepare scheduling and platform matrix.
3. Prepare engagement rules.
4. Apply approval gates for publishing where required.
5. Prepare social packet.

## PHASE-7 / STEP-1 : Attribution
1. Map content to platform.
2. Map platform to traffic.
3. Map traffic to Shopify outcome.
4. Map payment/deposit outcome to Stripe when available.
5. Prepare attribution ledger fields.
6. Prepare optimization feedback loop.

## PHASE-8 / STEP-1 : Validation
1. Validate workflow completeness.
2. Validate source truth.
3. Validate governance classification.
4. Validate stack dependencies.
5. Validate rollback path.
6. Validate continuity state.
7. Produce validation report.

## PHASE-9 / STEP-1 : Escalation
1. Log blocker.
2. Identify cause.
3. Attempt workaround.
4. Attempt self-heal.
5. Escalate only after governed paths are exhausted.
6. Record human-needed state.

## PHASE-10 / STEP-1 : Dehydrate / Sign-Out
1. Record what was done.
2. Record what changed.
3. Record tests run.
4. Record blockers.
5. Record workarounds.
6. Record self-heal result.
7. Record current phase and step.
8. Generate exact NEXT GPT INSTRUCTION.

## Governance Rules
1. Use sandbox-first execution.
2. Preserve Drive/Sheets-first canon.
3. Preserve governance mutation lock.
4. End every operational response with executive final block.
5. MCP/OpenAPI/SSE remains parked unless Jeremy explicitly reopens it.
