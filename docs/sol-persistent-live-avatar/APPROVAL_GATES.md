# SOL Approval Gates

## Purpose
Define approval checkpoints for SOL Persistent Live Avatar Assistant v1.

## Required Human Approval
- Production deployment
- Production SQL
- Billing changes
- Public social posting
- Shopify live mutations
- Mass outbound messaging
- Destructive rollback actions

## Automatic Allowed Actions
- Sandbox validation
- Static analysis
- Local build checks
- Draft generation
- Staging planning
- Logging and reporting

## Approval Workflow
1. AUTO BUILDER prepares action.
2. Policy gate evaluates risk.
3. Action logged in execution ledger.
4. Human approval requested when required.
5. Mutation proceeds only after approval evidence is captured.
