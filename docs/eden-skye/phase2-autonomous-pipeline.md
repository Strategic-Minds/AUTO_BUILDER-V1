# Eden Skye Studios Phase 2 Autonomous Pipeline

## Objective

Build a draft-only autonomous operating system for Eden Skye Studios that can plan, generate, review, approve, and optimize luxury social content while routing commerce attention back to `edenskyestudios.com` through Shopify.

## Current Scope

This branch documents the Phase 2 setup-only implementation. It is intentionally non-publishing and non-deploying.

Enabled for setup:

- Supabase campaign, approval, social draft, media job, Shopify sync placeholder, and receipt records.
- GitHub source-of-truth documentation and draft PR review.
- Google Drive operating docs and trackers where connector support allows.
- Notion command-center databases only when a verified parent is available.
- Google Calendar operating blocks only when calendar write tools are available.

Disabled until separate approval:

- Shopify mutations.
- Vercel production deploys.
- Xyla autoposting.
- Live social publishing.
- Payment, pricing, discount, or subscription changes.

## Brand Frame

Eden Skye Studios is a luxury AI lifestyle membership ecosystem with a black, gold, champagne, marble, satin, candlelight, cinematic-night visual system. The primary commercial path is the Eden Skye Black Card Membership.

The brand voice should remain:

- premium and composed
- warm, feminine, and emotionally intelligent
- cinematic and exclusive
- platform-safe and commercially usable
- clearly fictional AI/virtual creator aligned, without deception

## Operating Stack

Shopify / `edenskyestudios.com`:

- primary website and commerce path
- Black Card Membership landing and waitlist funnel
- traffic destination for approved campaigns
- no mutation until separate approval

Xyla:

- future connected autoposting or channel execution lane
- must remain disabled until publishing approval
- receives only approved, platform-safe, final packets

Higgingfield.ai:

- external creative production lane
- draft visuals and content concepts only until asset approval

HeyGen:

- avatar and spoken-video production lane
- uses approved scripts only

Notion:

- command center, campaign databases, approval tracking, SOPs

Google Drive:

- working documents, creative packets, asset specs, approval checklists

Google Calendar:

- content operating blocks, review windows, weekly optimization loops

Supabase:

- campaign records, draft queue, approval requests, receipts, tool logs, sync events

GitHub / Auto Builder:

- governed source-of-truth docs, pipeline specs, implementation packets, PR review

## Automation Logic

The automation target is governed autonomy, not uncontrolled mutation.

Daily loop:

1. readiness check
2. trend and campaign context scan
3. draft packet generation
4. media prompt/script preparation
5. approval request creation
6. draft queue update
7. receipt logging

Weekly loop:

1. metric review
2. winning pattern identification
3. prompt/script refinement
4. content calendar adjustment
5. funnel and traffic recommendation update

## Safety Boundary

The system must never promise adult access, never publish explicit content, never use private assets without approval, and never publicly post or mutate live commerce infrastructure without a separate approval event.
