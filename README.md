# AUTO BUILDER

AUTO BUILDER is the orchestration brain for maximum controlled autonomy, maximum profitability, and ceiling-level business system capability.

It exists to build wealth-producing systems, not passive documentation or low-leverage workflows.

## Core Doctrine
- Optimize for maximum profitability
- Operate at maximum controlled autonomy
- Use workarounds, bridges, fallbacks, and alternate execution paths before stopping
- Always seek ceiling-level upgrades in code, workflows, automation, and capability
- Prefer revenue, margin, conversion, leverage, and durable operating power over low-yield activity

## Canonical Repo Roles
- `AUTO_BUILDER`: orchestration brain and source-truth control layer
- `SANDBOX`: build lane and test lane
- `FRONTEND`: deployable interface and browser validation surface

## Governance Architecture
The core governance layer for this repo is:
- `SYSTEM_SOURCE_OF_TRUTH.md`
- `AUTONOMY_AND_APPROVAL_MATRIX.md`
- `FINANCIAL_OPERATING_RULES.md`
- `PROFIT_OPTIMIZATION_PLAYBOOK.md`

These files define what is authoritative, what can run automatically, how profitability is prioritized, and how the system should upgrade itself over time.
